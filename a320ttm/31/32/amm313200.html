    ATA-31-CFDS

    330吴彦祖 2023/3/14 指示/记录系统A320FM

    • 核心组件： 1个CFDIU(Centralized Fault Display Interface Unit，中央故障显示接口组件计算机)
    • 核心作用： 用于飞机维护
    • 人机交互接口组件：MCDU(参考ATA 22-自动飞行系统)
    • 额外功能： CFDIU提供备用时钟
    • 选装功能(接口)： 将维护数据通过ACARS发送回运营人的地面服务器

    # 一、简述

    飞机的绝大多数系统(的计算机)都与CFDIU相连，每个系统都有Build In Test Equipment(BITE)， BITE会将本系统的维护数据传输给CFDIU，CFDIU采集到这些维护数据后会对数据进行二次处理， 最后可以在MCDU上进行操作，来查看这些数据，或通过机载打印机打印出来。

    因此，Centralized Fault Display System(CFDS)的核心功能由一部CFDIU(处理数据)和飞机各个系统的BITE(产生数据)来完成， 除此之外，MCDU、打印机和ACARS也是组成CFDS的一份子

    # 二、BITE

    飞机上每一个电子或电气系统都装有Build In Test Equipment，BITE是一种软硬件一体的设备(可以理解为嵌入式计算机系统)， 它是飞机系统的一部分，用来检测、监测、定位、存储本系统的故障

    # 1、工作原则

    • 区分并存储本系统的内部故障和外部故障(根据系统不同可以存储一个或多个航段)

    • 内部故障：本系统自身的软硬件引起的故障
    • 外部故障：不是所有的系统都是独立，很多系统需要和其他系统交互，需要其他系统的数据才能完成自身的功能，由其他系统原因导致的本系统工作不正常视为外部故障
    • 在飞行中存储下来的故障数据不受地面维护操作的影响
    • 分区存储，飞行中和地面上的故障存储在不同的区域(好比电脑的硬盘分区或装了多个硬盘的电脑)
    • 监控本系统所有LRU的输入总线(ARINC 429)
    • 间歇性的故障也会存储
    • 使用ATA章节号或FIN号来标识故障
    • 在飞行中，一直保持实时传输维护信息

    # 2、故障的三种类别(Category)

    根据故障的重要程度，BITE将故障分为三种类别

    • Class 1： 非常重要，需要立即处理

    在飞行中Class 1故障会立即告知机组：比如通过音响或灯光警告、ECAM上的信息

    • Class 2： 一般，不需要立即处理，但再次发生的话就需要处理了

    Class 2不会在飞行中告知机组，在落地双发关车后，通过维护信息(SD的状态页面)告知

    • Class 3： 不重要，因为不影响安全性和适用性，闲得没事才去管

    Class 3在地面和飞行中都不会展示，只能手动去相应系统的页面里找看有没有

    # 三、系统的三种类型(Type)

    与故障分类相似，连接到CFDS的各个系统也分为三种类型

    # 1、一型系统 Type 1

    • I/O总线都是ARINC 429，且输入总线只用于连接CFDS
    • (BITE)最多可存储64个航段的数据
    • (BITE)分5个存储区
      • Zone 1，大概能存储30条左右故障：
        • 存储在当前航段或最近一个航段故障的LRU的标识
        • 存储最近63个航段故障的LRU的标识(实际上后台超过64个航段，但只存储最近63个)
      • Zone 2(Trouble Shooting Data)，大概能存储30条左右故障：
        • 存储最近64个航段发生的故障的补充信息(例如故障时飞机的构型状态)
      • Zone 3(Trouble Shooting Data)，大概能存储3条左右故障：
        • 存储在地面短停期间发生的故障(LRU的标识和相应的补充数据)
      • Zone 4
        • 存储故障的详细信息，无法通过MCDU查看，必须拆下计算机使用专用的设备读取
      • Zone 5，存储Class 3故障，大概能存储400FH

    # 2、二型系统 Type 2

    • ARINC 429输出，离散输入
    • (BITE)只记录最后一个航段的数据
    • (BITE)分4个存储区
      • Zone 1，当前或最近一个航段的故障
      • Zone 2，当前或最近一个航段的故障的补充数据，只能拆机读取
      • Zone 3，最近一个短停期间发生的故障
      • Zone 4，故障的详细信息，只能拆机读取

    # 3、三型系统 Type 3

    • I/O都是离散的
    • (BITE)只记录当前航段的数据
    • (BITE)没有存储(掉电数据就没了，相当于只有内存，没有硬盘)

    Type 3系统的输入只能用于启动地面测试或系统重置
