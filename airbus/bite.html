    BITE

    330吴彦祖 2023/4/6 指示/记录系统

    ATA-31

    BITE是Built-In-Test Equipment的缩写，是一种用来检测和隔离故障的装置，它是空客飞机中央故障显示系统功能的重要组成之一。

    # 一、简介

    BITE是一种软硬件结合的电子设备（嵌入式）。

    根据空客的飞机设计原则，在空客的飞机上，所有电气（electrical）和电子（electronic）系统都有BITE功能。 BITE是这些系统或计算机不可分割的一部分，它的核心作用就是自检，即通过执行测试来定位、隔离和存储本系统的故障或故障的部件。 BITE测试可以人工启动，也可以是自动的（在飞行中有些计算机就会不停的执行BITE测试来监控系统状态）。

    内部和外部LRU

    假设某个系统由多个LRU组成，且这些LRU都专属于该系统，则称这些LRU是该系统的内部LRU；

    但更多的情况下，系统与系统之间是要交流的，需要数据传递的，当该系统需要的数据中有来自于其他系统时，其他系统可视为该系统的传感器，则称其他系统或其对应的LRU为外部LRU。

    例如：在自动驾驶系统中，它的控制组件、算法计算机就是自动驾驶系统的内部LRU；而自动驾驶系统需要的大气数据、惯导数据、无线电高度表等等都是外部LRU

    一个LRU只能划分为某一个系统的内部LRU，不能同时作为多个系统的内部LRU，例如很多系统都会用到压力传感器，但压力传感器只能作为其中一个系统的LRU

    所有具有BITE功能的系统都会通过系统总线直接连接到CFDIU

    但有两个例外：

    • FMGS通过FAC-1的总线连接到CFDIU
    • EFCS通过FCDC的总线连接到CFDIU

    故障的监测和隔离由相应系统自己负责，CFDS不会做任何处理，也不会修改系统的诊断结果

    对于复杂的系统而言，由其众多计算机中的某一部担任系统BITE的角色，也就是说它负责收集来自外围设备计算机的维护数据，并保障系统中的计算机们与CFDS之间的接口

    # 二、BITE的工作原则

    BITE有两个工作模式：正常模式（自动）和菜单模式（手动）

    • 不管是内部LRU还是外部LRU，只要发生故障会影响到本系统工作的，BITE都会检测
    • BITE存储并区分内部和外部故障（根据不同系统，有的能存储一个航段，有的能存储多个航段）
    • 在飞行中存储的故障数据不会被各种各样的地面维护工作所影响
    • BITE在空中和地面都能存储故障数据，但存储区域不同
    • 根据每个系统的配置不同，所有LRU的输入总线都会被监控（LRU的输入总线都是ARINC 429总线）
    • 间歇性的故障在经过几个周期的确认后才会被存储
    • 故障的LRU有时通过FIN号来标识，有时通过ATA 100章节号来标识
    • 在飞行中，BITE会一直实时传输维护相关的信息，这些信息涵盖当前航段或最近一个航段的内部和外部故障

    # 三、BITE对故障的分类

    根据对飞机安全性和使用性的影响程度，BITE将故障分为三类管理：

    • Class 1
    • Class 2
    • Class 3

    Class 1和Class 2类的主故障、独立故障的警告信息会通过FWC实时发送给CFDIU，CFDIU会存储下来并生成ECAM报告

    # Class 1(紧急)

    影响程度最严重，直接影响当前飞行的安全，在空中时，会产生音响警告，会在ECAM上显示相关故障信息，如果该系统还连接到FWC，FWC会直接收到故障信息，从而产生警告

    # Class 2(不急)

    不会影响当前或后续的飞行安全，但是，在此基础上再发生一个故障时，就会影响飞行安全了，当飞机落地发动机关车后，Class 2故障信息会在SD的全局状态页面以维护状态信息的形式显示给机组看

    # Class 3(不用管)

    Class 3的故障重要程度最低，因为它不会对飞机的安全性和使用性产生后果。在日常维护中不用管，除非闲得没事做，它甚至都不会显示出来让人看，你得主动去CFDS里查找看是否有Class 3故障
